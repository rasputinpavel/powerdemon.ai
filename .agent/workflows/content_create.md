# Создание контента

Ты — контент-мейкер. Задача — создать готовый к публикации контент и положить его в `drafts/` на одобрение оператору.

## Вход

- Прочитай `context.md` в папке бизнеса
- Прочитай `knowledge/instagram_strategy_*.md` (контент-план, стиль, идеи)
- Прочитай `knowledge/yandex_direct_*.md` (если создаём рекламные кампании)
- Посмотри `drafts/_queue.md` — что уже создано, чтобы не дублировать

## Шаг 1: Определить, что создаём

Спроси оператора или определи из контент-плана:
1. Канал: Instagram / Яндекс.Директ
2. Формат: пост / карусель / Reels / объявление
3. Тема: из контент-плана или по запросу
4. Количество: сколько единиц контента создать

## Шаг 2: Создать контент

### Для Instagram-поста/карусели:
1. Написать текст подписи (caption) с emoji и CTA
2. Подобрать хештеги (10-15 релевантных)
3. Сгенерировать визуал через `generate_image`:
   - Описать промпт на основе визуального стиля из стратегии
   - Генерировать в формате, подходящем для Instagram (1:1 или 4:5)
4. Для карусели: создать несколько слайдов

### Для Reels:
1. Написать полный сценарий (хук → основная часть → CTA)
2. Написать тексты на экране (3-5 фраз)
3. Сгенерировать обложку через `generate_image`
4. Описать инструкцию по съёмке (что снять, какой ракурс)

### Для Яндекс.Директ:
1. Написать заголовки (56 символов + 30 символов)
2. Написать тексты объявлений (81 символ)
3. Быстрые ссылки (4 штуки)
4. Уточнения (4 штуки)

## Шаг 3: Сохранить черновик

Создать папку в `businesses/{business_name}/drafts/{channel}/{date}_{slug}/`:

**Для Instagram:**
```
drafts/instagram/{date}_{slug}/
├── caption.md        # текст + хештеги + CTA
├── visual_1.webp     # сгенерированное изображение
├── visual_2.webp     # (для карусели)
└── meta.md           # формат, рекомендуемое время публикации
```

**Для Яндекс.Директ:**
```
drafts/yandex_direct/{date}_{slug}/
├── campaign.md       # все объявления, ключи, настройки
└── meta.md           # бюджет, гео, даты
```

## Шаг 4: Обновить очередь

Добавить запись в `businesses/{business_name}/drafts/_queue.md`:

```markdown
| # | Дата | Канал | Контент | Статус |
|---|------|-------|---------|--------|
| N | {date} | {channel} | {описание} | ⏳ На одобрении |
```

## Шаг 5: Уведомить оператора

Покажи оператору превью созданного контента:
- Текст подписи
- Превью визуала
- Рекомендуемое время публикации

Спроси: «Одобряешь? Или нужны правки?»

После одобрения обнови статус в `_queue.md` на ✅.
