# ICP-анализ

## Вход

- Прочитай `context.md` в папке бизнеса
- Прочитай имеющиеся файлы в `knowledge/` (исследования рынка, если есть)

## Шаг 1: Собрать данные о текущих/потенциальных клиентах

Спроси оператора:
1. Кто сейчас покупает (или кому хочется продавать)?
2. Есть ли данные о существующих клиентах?
3. Какие проблемы решает продукт?
4. Кто готов платить больше всех?

## Шаг 2: Определить сегменты

Для каждого потенциального сегмента заполни:

```markdown
### Сегмент: {название}
- **Кто:** демография, должность, размер компании
- **Боль:** главная проблема
- **JTBD:** какую работу нанимают продукт выполнить
- **Готовность платить:** высокая / средняя / низкая
- **Доступность:** легко / средне / сложно достучаться
- **Каналы:** где их искать
- **Оценка привлекательности:** ⭐⭐⭐⭐⭐ (1-5)
```

## Шаг 3: Приоритизировать сегменты

Составь матрицу:

```markdown
| Сегмент | Размер | Боль | Готовность платить | Доступность | Итого |
```

Ранжируй от самого перспективного к наименее.

## Шаг 4: Сформулировать ICP

Для топ-1-2 сегментов составь детальный профиль:

```markdown
## Ideal Customer Profile

**Кто:** ...
**Возраст:** ...
**Должность/роль:** ...
**Размер компании:** ...
**Боль:** ...
**Триггер покупки:** что заставляет начать искать решение
**Возражения:** что может остановить
**Критерии выбора:** по чему выбирают между решениями
**Где обитают:** каналы, сообщества, медиа
```

## Шаг 5: Сохранить результат

Файл: `businesses/{business_name}/knowledge/icp_{YYYY-MM-DD}.md`

Обнови `log.md` текущего проекта.
