# Еженедельный отчёт

## Вход

- Прочитай `context.md` в папке бизнеса
- Прочитай `log.md` всех активных проектов
- Прочитай последние карточки экспериментов (если есть)

## Шаг 1: Собрать данные

Для каждого активного проекта:
1. Что было сделано за неделю? (из `log.md`)
2. Какие метрики изменились?
3. Есть ли блокеры или проблемы?
4. Что запланировано на следующую неделю?

Спроси оператора: есть ли что-то, что нужно добавить? Метрики из внешних систем?

## Шаг 2: Сформировать отчёт

Используй шаблон `_templates/weekly_report.md`:

```markdown
# Еженедельный отчёт: {BUSINESS_NAME}
Период: {дата_от} — {дата_до}

## Что сделано
- Проект A: ...
- Проект B: ...

## Ключевые метрики
| Метрика | Прошлая неделя | Эта неделя | Δ |
|---------|---------------|------------|---|

## Проблемы и блокеры

## План на следующую неделю

## Расход бюджета
```

## Шаг 3: Сохранить отчёт

Файл: `businesses/{business_name}/projects/{project}/assets/report_{YYYY-MM-DD}.md`

Обнови `log.md` текущего проекта.

## Шаг 4: Показать оператору

Покажи краткую сводку оператору и спроси:
- Всё ли корректно?
- Нужно ли что-то дополнить?
- Нужно ли отправить клиенту? (в будущем — Telegram-интеграция)
