# Онбординг нового клиента

## Шаг 1: Собрать информацию

Задай оператору вопросы:

1. **Название бизнеса** (латиницей, snake_case — будет именем папки, например `vasya_shop`)
2. **Ниша / индустрия**
3. **Продукт или услуга** — что продают?
4. **Сайт** (если есть)
5. **География** — где работают?
6. **Целевая аудитория** — кто покупает? (кратко)
7. **Текущая стадия** — запуск, рост, зрелость?
8. **Маркетинговый бюджет** (если известен)
9. **Что нужно в первую очередь?** — какая задача самая приоритетная?

## Шаг 2: Создать структуру папок

Создай следующую структуру:

```
businesses/{business_name}/
├── context.md
├── projects/
└── knowledge/
```

## Шаг 3: Заполнить context.md

Скопируй шаблон из `_templates/business_context.md` в `businesses/{business_name}/context.md` и заполни его ответами из Шага 1.

## Шаг 4: Обновить реестр

Добавь новую строку в `businesses/_index.md` с данными:
- Название бизнеса
- Ниша
- Статус: `активный`
- Дата онбординга: сегодня
- Активные проекты: —

## Шаг 5: Предложить следующий шаг

На основе ответа на вопрос 9 предложи оператору запустить подходящий скилл:
- Если нужна реклама → `/ad_campaign_yandex` или `/ad_campaign_google`
- Если нужно исследование → `/market_research`
- Если нужно определить аудиторию → `/icp_analysis`
- Если нужен выход на рынок → `/new_market_entry`
