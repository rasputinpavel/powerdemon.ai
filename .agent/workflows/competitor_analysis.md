# Анализ конкурентов

## Вход

- Прочитай `context.md` в папке бизнеса
- Определи продукт, нишу и географию

## Шаг 1: Определить конкурентов

Используй веб-поиск чтобы найти:
- **Прямые конкуренты** (5-10) — предлагают похожий продукт
- **Косвенные конкуренты** (3-5) — решают ту же проблему иначе

Спроси оператора: знает ли он конкурентов, которых обязательно включить?

## Шаг 2: Собрать данные по каждому

Для каждого конкурента через веб-поиск и посещение сайтов определи:

1. **Продукт** — что продают, основные фичи
2. **Цены** — тарифы, модель монетизации
3. **Позиционирование** — как себя подают, УТП
4. **Каналы продвижения** — где рекламируются, соцсети, контент
5. **Сильные стороны**
6. **Слабые стороны / отзывы клиентов**
7. **Примерный трафик** (если можно определить)

## Шаг 3: Составить сравнительную таблицу

```markdown
| Конкурент | Продукт | Цена | УТП | Каналы | Слабости |
|-----------|---------|------|-----|--------|----------|
```

## Шаг 4: Выводы и возможности

- Что делают все, но плохо?
- Какие сегменты не обслуживают?
- В чём наше потенциальное преимущество?
- Какие каналы не используют?

## Шаг 5: Сохранить результат

Файл: `businesses/{business_name}/knowledge/competitors_{YYYY-MM-DD}.md`

Обнови `log.md` текущего проекта.
