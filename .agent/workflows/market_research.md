# Deep Research рынка

## Вход

- Прочитай `context.md` в папке бизнеса
- Определи нишу, продукт и географию

## Шаг 1: Определить рамки исследования

Уточни у оператора (или определи из контекста):
1. Какой рынок исследуем? (ниша + география)
2. Есть ли конкретные вопросы, на которые нужен ответ?
3. Какой уровень глубины нужен? (обзор / детальный)

## Шаг 2: Собрать данные

Используй веб-поиск для сбора информации по следующим блокам:

### 2.1 Размер рынка
- TAM / SAM / SOM (если применимо)
- Объём рынка в деньгах и клиентах
- Темпы роста

### 2.2 Тренды
- Ключевые тренды в индустрии
- Технологические изменения
- Регуляторные изменения

### 2.3 Ключевые игроки
- Топ-5-10 компаний
- Их доля рынка (оценочно)
- Основные продукты

### 2.4 Потребители
- Кто покупает?
- Как принимают решения?
- Основные каналы

### 2.5 Возможности
- Недообслуженные сегменты
- Пробелы в рынке
- Точки входа для нашего бизнеса

## Шаг 3: Оформить отчёт

Сохрани результат в `businesses/{business_name}/knowledge/market_research_{YYYY-MM-DD}.md`

Структура отчёта:
```markdown
# Исследование рынка: {рынок}
Дата: {date}

## Резюме (3-5 предложений)
## Размер рынка
## Тренды
## Ключевые игроки (таблица)
## Потребители
## Возможности для {business_name}
## Источники
```

## Шаг 4: Обновить лог проекта

Добавь запись в `log.md` текущего проекта о проведённом исследовании со ссылкой на файл.
